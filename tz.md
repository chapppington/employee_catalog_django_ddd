# Тестовое задание для junior python-разработчика

## Описание

Вам нужно разработать онлайн-каталог сотрудников для компании, чей штат насчитывает больше 50 000 человек.

---

## Часть № 1 (обязательная)

Создайте веб-страницу, которая будет выводить иерархию сотрудников в древовидной форме.

### Требования к данным о сотрудниках

Информация о каждом сотруднике должна храниться в базе данных и содержать следующие данные:

- **ФИО**
- **Должность**
- **Дата приёма на работу**
- **Размер заработной платы**

### Дополнительные требования

- У каждого сотрудника есть 1 начальник
- База данных должна содержать не менее 50 000 сотрудников и 5 уровней иерархий
- Не забудьте отобразить должность сотрудника

---

## Часть № 2 (опциональные пункты)

### 1. Миграции и база данных

- Создайте базу данных, используя миграции Django / Flask
- Используйте DB seeder для Django ORM / Flask-SQLAlchemy для заполнения базы данных

### 2. Стилизация

- Используйте Twitter Bootstrap для создания базовых стилей вашей страницы

### 3. Список сотрудников

- Создайте ещё одну страницу и выведите на ней список сотрудников со всей имеющейся о них информацией из базы данных
- Реализуйте возможность сортировки по любому полю

### 4. Поиск

- Добавьте возможность поиска сотрудников по любому полю для страницы, созданной в пункте 3

### 5. AJAX функциональность

- Добавьте возможность сортировать и искать по любому полю без перезагрузки страницы, например используя AJAX

### 6. Аутентификация

- Используя стандартные функции Django / Flask, осуществите аутентификацию пользователя для раздела веб-сайта, доступного только для зарегистрированных пользователей

### 7. Защищённый раздел

- Перенесите функционал, разработанный в пунктах 3, 4 и 5 (используя AJAX запросы) в раздел, доступный только для зарегистрированных пользователей

### 8. CRUD-операции

- В разделе, доступном только для зарегистрированных пользователей, реализуйте остальные CRUD-операции для записей сотрудников
- Все поля, касающиеся пользователей, должны быть редактируемыми, включая начальника каждого сотрудника

### 9. Загрузка фотографий

- Осуществите возможность загружать фотографию сотрудника и отобразите её на странице, где можно редактировать данные о сотруднике
- Добавьте дополнительную колонку с уменьшенной фотографией сотрудника на странице списка всех сотрудников

### 10. Перераспределение сотрудников

- Осуществите возможность перераспределения сотрудников в случае изменения начальника
- Бонусом может быть то, что вы сможете это осуществить с применением встроенных механизмов/парадигм, предлагаемых Django ORM / Flask-SQLAlchemy ORM

### 11. Ленивая загрузка

- Реализуйте ленивую загрузку для дерева сотрудников
- Например, показывайте первые два уровня иерархии по умолчанию и подгружайте 2 следующих уровня или всю ветку дерева при клике на сотрудника второго уровня

### 12. Drag-n-drop

- Реализуйте возможность менять начальника сотрудника используя drag-n-drop сразу в дереве сотрудников
